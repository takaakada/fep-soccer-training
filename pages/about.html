📚 指導理論

Active Training の理論体系と設計思想

💡
Active Training は自由エネルギー原理（FEP）に基づくサッカー育成プログラムです。
「予測して、ズレを知り、修正する」 — この脳の基本原理をトレーニングに応用します。
🧠 基本思想 — 脳は予測する機械
脳は常に「こうなるはず」という予測（内部モデル）を持ち、実際の感覚入力と比較しています。
この予測と現実のズレが「予測誤差」であり、脳はこのズレを最小化しようと学習し続けます。
サッカーにおける「上手さ」とは、この予測精度と修正速度の高さです。

📐

⚡

📊 4つの評価指標（スコア体系）

選手の状態を4つの数値で定量化し、トレーニングの設計と振り返りに活用します。

VFE 変分自由エネルギー
「今のプレーのズレ」を測る指標。
練習の難易度（complexity）と成功率（accuracy）から算出。
低い = 安定、高い = 混乱
F = wc · sigmoid(難易度) + wa · sigmoid(成功率)
→ 0〜100 のスコア表示
EFE 期待自由エネルギー
「将来の不確実性」を測る指標。月1回記録。
認知面（epistemic）と実利面（pragmatic）の2軸。
α（探索重み）とβ（実利重み）の変化を追跡。
EFE = -α · tanh(epistemic) - β · tanh(pragmatic)
→ 0〜100 のスコア表示
F' 感情反応性
「感情がプレーにどう影響するか」を測る指標。
成長実感（dr）・満足感（uh）・次回期待（fh）の3要素。
正 = ポジティブ、負 = ネガティブ
F' = 0.25·Q1 + 0.50·Q2 + 0.25·Q3
反応性 = 1 + 0.3·λ + 0.5·u
EU 推定不確実性
「自己認識のブレ幅」を示す統合指標。
VFE × F' × σ を統合し、選手の全体的な不確実性を算出。
長期的な安定化傾向を追跡。
EU = sigmoid(F') + σ_shift
σ更新: ln(σnew) = ln(σ) + 0.5 · F'
🧬 神経修飾系マッピング

脳内の3つの神経伝達物質の役割を、トレーニングパラメータに対応させています。

🔵 セロトニン → σ（精度重み）
コンディション・身体状態に連動。
良好 → σ↑ → 精度への感度が上がる
不調 → σ↓ → 誤差の影響を受けやすい
入力: 前チェックの「コンディション」
🟣 オキシトシン → λ（感情反応性）
安心感・信頼感に連動。
安心型声かけ → λ↓ → 感情揺れが小さくなる
不安・孤立 → λ↑ → 感情揺れが大きくなる
入力: コーチ記録の「声かけタイプ」
🟡 ドーパミン → F'（感情価）
期待感・楽しさ・成長実感に連動。
楽しい・成長実感 → F'↑ → ポジティブ
退屈・挫折感 → F'↓ → ネガティブ
入力: 前チェックの「期待感」+ セッション後振り返り
🏗️ 4つのトレーニング層（Layer 構造）

単純な個人の感覚から実戦的な複雑状況まで、4層構造でトレーニングの難易度を設計します。

L1 🧠 単純（個人）
  • 自分の身体・動作のズレを知る
  • 動作の予測と実測を比較する
  • 単純な反復で感覚を研ぎ澄ます
  • コンディションの変化を認識する
L2 🤝 応用（対人）
  • 味方・相手の動きを予測する
  • チーム内でイメージを共有する
  • ミラーリング・身体同調を育てる
  • 判断を伴う応用的な課題に取り組む
L3 🌊 複合（適応）
  • 複数の要素を同時に処理する
  • ルール変更・状況変化に対応する
  • あえて予測が外れる状況を作る
  • 変化への対応速度を高める
L4 ⚡ 実戦（統合）
  • 試合に近い強度・速度で実施する
  • プレッシャー下での判断維持
  • 予測が外れても崩れない力
  • 全レイヤーの統合と再現性
⚠️ 11の問題構造（Problem Master）

選手が直面するあらゆるエラーは、以下の11の問題構造に分類されます。
個別トレーニングプランはこれらの問題の組み合わせに基づいて設計されます。

P1 予測はズレる
P2 感覚は不正確
P3 情報が多すぎる
P4 反応は遅い
P5 どこを直すか分からない
P6 1つ変えると他も変わる
P7 状況変化で更新が必要
P8 複雑さを単純化しにくい
P9 報酬・コストが行動を左右する
P10 感情変化が邪魔する
P11 他者と完全理解できない
🏃 5つのエラーパターン（個別トレーニング分類）

11の問題構造から導出された5つの典型パターン。選手のタイプを分類し、個別のトレーニングプランを提案します。
各プランはGoogleスプレッドシートで管理され、アプリに自動反映されます。

⏱️
IND-001 反応遅れ型 主問題: P4（反応は遅い）
見てから動くため初動が遅れる → 予測準備を鍛えるプラン
🌀
IND-002 情報過多型 主問題: P3（情報が多すぎる）
情報が多いと判断が止まる → 優先情報を絞るプラン
🔄
IND-003 修正迷子型 主問題: P5（どこを直すか分からない）
ミス後に何を直せばいいか分からない → 修正の優先順位を整理するプラン
🧊
IND-004 変化停止型 主問題: P7（状況変化で更新が必要）
状況変化で思考と動きが止まる → 予測更新の速度を鍛えるプラン
💔
IND-005 感情引きずり型 主問題: P10（感情変化が邪魔する）
ミス後に次のプレーへ移れない → 再安定化を身につけるプラン

📏 難易度設計の考え方
成功率を目安に適切な練習負荷を設計します。
予測誤差が大きすぎると学習が進みません。
逆に小さすぎると成長が止まります。
✅ 成功率 70% — 安定の基準。再現性を確認する。
🔸 成功率 50% — 修正すれば届く範囲。学習効果が最も高い。
🎯 成功率 30% — 挑戦的な課題。取り組む意味がある難しさ。
⚠️ 成功率 10%以下 — 負荷設定の見直しが必要。

👥 3つの年代別アプローチ

発達段階に応じてトレーニングの重点を変えます。

🟦 〜 小学生
  • 楽しさを最優先にする
  • 身体感覚を豊かにする遊び
  • 仲間とのつながりを育てる
  • ミスを怖がらない雰囲気作り
  • 失敗→修正を遊びとして体験する
重点: L1（個人の感覚）+ 楽しさ
🟩 中〜高校生
  • 自分のレベルを認識する
  • 技術の再現性を高める
  • 判断の切り替え速度を上げる
  • 制約下での適応力を高める
  • 振り返りと言語化を練習する
重点: L2（チーム）+ 言語化
🟧 大学生・社会人
  • 微細な誤差修正を行う
  • 高速変化の中での意思決定
  • 高強度ストレス下での安定性
  • チーム共有モデルの構築
  • 自己とチームの評価と調整
重点: L3（適応）+ 統合
🗣️ コーチングのフィードバック指針
💡
原則：「正解を与える」のではなく「何を予測していたか」を聞く。
選手の内部モデルを可視化し、アップデートを助ける。
🗣️ 問いかけ方
  • 何を見てそう判断した?
  • どこで予想と違った?
  • 自分の動きはどうだった?
  • 次は何を意識する?
🔍 エラーの分類
  • 予測全体がズレていた
  • 感覚・身体の動きが悪かった
  • 状況把握がうまくできなかった
  • チーム連携がうまくいかなかった
📈 選手と指導者のズレを修正する
  • 選手が感じたズレを共有する
  • 指導者の意図と選手の理解を擦り合わせる
  • 選手の「どうなりたいか」を理解する
  • 結果ではなく次の1プレーを評価する
